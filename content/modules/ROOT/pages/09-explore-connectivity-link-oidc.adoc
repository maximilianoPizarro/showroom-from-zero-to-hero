= OIDC Authentication (Neuralbank)

In this module you will explore how **Red Hat Connectivity Link** protects the pre-deployed **Neuralbank** stack using **OIDC** (OpenID Connect via Keycloak) and **RateLimitPolicy**.

== Architecture

The Neuralbank stack in the `neuralbank-stack` namespace:

[cols="1,2"]
|===
| Component | Description

| `neuralbank-backend` | Quarkus REST API exposing `/api/v1/customers`
| `neuralbank-db` | PostgreSQL database with seed data
| `neuralbank-frontend` | Web interface for credit visualization
| `neuralbank-gateway-istio` | Istio ingress gateway (auto-managed by Gateway API)
|===

[mermaid]
....
sequenceDiagram
    participant Client
    participant Gateway as Istio Gateway
    participant Auth as Kuadrant AuthPolicy
    participant RL as RateLimitPolicy
    participant Backend as Neuralbank Backend

    Client->>Gateway: GET /api/v1/customers
    Gateway->>Auth: Validate JWT
    alt No Token
        Auth-->>Client: 302 Redirect to Keycloak
    else Valid Token
        Auth->>RL: Check rate limit (60 req/min)
        RL->>Backend: Forward request
        Backend-->>Client: 200 JSON response
    end
....

== Inspect the Gateway and HTTPRoute

[.console-input]
[source,bash]
----
oc get gateway -n neuralbank-stack
oc get httproute -n neuralbank-stack
----

The Gateway should show `PROGRAMMED: True`. The HTTPRoute forwards `/api` and `/q` paths to the backend.

== Verify the AuthPolicy

[.console-input]
[source,bash]
----
oc get authpolicy -n neuralbank-stack -o yaml
----

The AuthPolicy links to the Keycloak realm `neuralbank` and protects the HTTPRoute:

[source,yaml]
----
apiVersion: kuadrant.io/v1
kind: AuthPolicy
metadata:
  name: neuralbank-backend-auth
  namespace: neuralbank-stack
spec:
  targetRef:
    group: gateway.networking.k8s.io
    kind: HTTPRoute
    name: neuralbank-backend-route
  defaults:
    rules:
      authentication:
        oidc:
          jwt:
            issuerUrl: https://rhbk.{cluster_domain}/realms/neuralbank
          credentials:
            authorizationHeader:
              prefix: Bearer
----

== Verify the RateLimitPolicy

[.console-input]
[source,bash]
----
oc get ratelimitpolicy -n neuralbank-stack -o yaml
----

[source,yaml]
----
apiVersion: kuadrant.io/v1
kind: RateLimitPolicy
metadata:
  name: neuralbank-backend-rlp
spec:
  targetRef:
    group: gateway.networking.k8s.io
    kind: HTTPRoute
    name: neuralbank-backend-route
  defaults:
    rates:
      - limit: 60
        window: 1m
----

== Test: Request Without Token (expect 302)

[.console-input]
[source,bash]
----
curl -s -o /dev/null -w "HTTP Status: %{http_code}\n" \
  https://neuralbank.{cluster_domain}/api/v1/customers
----

[.console-output]
----
HTTP Status: 302
----

The AuthPolicy redirects unauthenticated requests to Keycloak login.

== Obtain a Bearer Token from Keycloak

[.console-input]
[source,bash]
----
TOKEN=$(curl -s -X POST \
  "https://rhbk.{cluster_domain}/realms/neuralbank/protocol/openid-connect/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=password" \
  -d "client_id=neuralbank-frontend" \
  -d "username=%USER_NAME%" \
  -d "password=Welcome123!" \
  | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")

echo "Token obtained (first 50 chars): ${TOKEN:0:50}..."
----

== How OIDC Authentication Works

[mermaid]
....
sequenceDiagram
    participant User
    participant Gateway as Istio Gateway
    participant Kuadrant as AuthPolicy
    participant Keycloak
    participant Backend

    User->>Gateway: Request without token
    Gateway->>Kuadrant: Check authentication
    Kuadrant-->>User: 302 Redirect to Keycloak
    User->>Keycloak: Login (username/password)
    Keycloak-->>User: JWT Access Token
    User->>Gateway: Request with Bearer token
    Gateway->>Kuadrant: Validate JWT signature + claims
    Kuadrant->>Backend: Forward authenticated request
    Backend-->>User: 200 JSON response
....

== Summary

The OIDC pattern is ideal for **interactive user-facing applications** where users authenticate through a web browser and tokens are issued by Keycloak. Rate limiting is applied per authenticated user.
