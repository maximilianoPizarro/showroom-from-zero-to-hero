= Explore Argo CD and APIs

This module connects the **GitOps** view in Argo CD with the **Software Catalog** in Developer Hub and **policies** applied in the cluster, so you understand each Neuralbank component end to end.

== Open the Argo CD UI

. Go to **Argo CD**: `https://openshift-gitops-server-openshift-gitops.{cluster_domain}`
. In **Applications**, filter by your namespace `{user_name}-neuralbank`.

You should see applications for **neuralbank-backend**, **neuralbank-frontend**, and **customer-service-mcp** (exact names depend on how the template registered the Application).

== Explore the resource tree

For each application:

. Click the name to open **Tree** or **Network** view.
. Observe **Deployments**, **Services**, **Routes**, **ConfigMaps**, **Secrets** (referenced, not values), and for MCP the **Gateway API** resources and policies.

Check **Sync Status**:

* *Synced* means the cluster matches the manifest in Git.
* *OutOfSync* means pending changes or drift; **Sync** applies desired state if you have permission.

Also review **Health**:

* *Healthy*, *Progressing*, and *Degraded* describe whether pods and routes are in the expected state.

NOTE: If a resource is unknown to Argo CD, the `argocd.argoproj.io` annotation may be missing or the resource may be managed by another Application.

== Sync and events

On the **Events** tab or detail view, review recent deployment messages. Useful when a new image fails to start or a sync hook fails.

== Developer Hub: component dependencies

. Open **Developer Hub** and open the **neuralbank-backend** entity card.
. Find **dependencies** or the relationship diagram (depending on installed plugins).
. Identify links to **APIs**, **systems**, or other **components** (frontend, MCP).

This complements Argo CD: Argo CD shows *resources*; the Hub shows *product intent* and ownership.

== API entity and OpenAPI

Many templates register an **API** entity alongside the **Component**:

. In the catalog, open the **API** entity associated with the backend (name aligned with the Neuralbank domain).
. Review the embedded or linked **OpenAPI** definition (as published in `catalog-info.yaml`).

The OpenAPI definition documents paths, schemas, and sometimes examples; it is the reference for service consumers and the frontend.

[.console-input]
----
Developer Hub -> Catalog -> API -> view OpenAPI / Swagger documentation
----

== Kuadrant policies on OpenShift

For the MCP service (and others exposed with the same pattern), inspect these objects in the console:

* **OIDCPolicy** — OIDC authentication requirements (integration with Keycloak or another IdP).
* **RateLimitPolicy** — rate limits applied to traffic matching the HTTPRoute.

Where they appear in the UI depends on installed CRDs; use **Search** in the console with the resource type if they are not in the sidebar.

NOTE: If you do not see policies, confirm Kuadrant CRDs are installed on the cluster. All resources should be in `{user_name}-neuralbank`.

== Summary

You correlated **GitOps state** in Argo CD with **catalog cards** in Developer Hub, located the **API** and its **OpenAPI**, and placed **OIDC and rate limit policies** in the Kuadrant context. This closes the loop between declarative deployment, living documentation, and edge security.
