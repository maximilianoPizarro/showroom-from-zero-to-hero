= Explore ArgoCD and APIs

This module connects the **GitOps** view in ArgoCD with the **Software Catalog** in Developer Hub and **policies** applied in the cluster.

== Open the ArgoCD UI

. Go to ArgoCD: `https://openshift-gitops-server-openshift-gitops.{cluster_domain}`
. In **Applications**, filter by your namespace `%USER_NAME%-neuralbank`.

You should see applications for **neuralbank-backend**, **neuralbank-frontend**, and **customer-service-mcp**.

== ArgoCD ApplicationSet

Applications are generated automatically by an **ApplicationSet** that watches Gitea for new repositories:

[source,yaml]
----
apiVersion: argoproj.io/v1alpha1
kind: ApplicationSet
metadata:
  name: workshop-neuralbank-backend
  namespace: openshift-gitops
spec:
  generators:
    - git:
        repoURL: https://gitea-gitea.{cluster_domain}
        revision: HEAD
        directories:
          - path: "*/neuralbank-backend"
  template:
    metadata:
      name: "{{path[0]}}-neuralbank-backend"
    spec:
      project: default
      source:
        repoURL: "https://gitea-gitea.{cluster_domain}/{{path[0]}}/neuralbank-backend.git"
        targetRevision: HEAD
        path: manifests
      destination:
        server: https://kubernetes.default.svc
        namespace: "{{path[0]}}-neuralbank"
----

== Explore the Resource Tree

For each application:

. Click the name to open **Tree** or **Network** view.
. Observe Deployments, Services, Routes, ConfigMaps, and for MCP the Gateway API resources and policies.

Check **Sync Status**:
* *Synced* — cluster matches the manifest in Git
* *OutOfSync* — pending changes or drift

Check **Health**:
* *Healthy*, *Progressing*, *Degraded*

== ArgoCD RBAC for Workshop Users

Workshop users can view their own applications in ArgoCD. The RBAC is configured via a ConfigMap:

[source,yaml]
----
apiVersion: v1
kind: ConfigMap
metadata:
  name: argocd-rbac-cm
  namespace: openshift-gitops
data:
  scopes: "[groups, preferred_username]"
  policy.csv: |-
    g, system:cluster-admins, role:admin
    # Each user sees their own applications
    p, user1, applications, get, default/user1-*, allow
    p, user1, applications, sync, default/user1-*, allow
    p, user1, logs, get, default/user1-*, allow
----

== API Entity and OpenAPI

Many templates register an **API** entity alongside the Component:

. In the catalog, open the **API** entity associated with the backend.
. Review the embedded **OpenAPI** definition.

The OpenAPI definition documents paths, schemas, and examples — the reference for service consumers and the frontend.

== Kuadrant Policies on OpenShift

For services exposed with the connectivity link pattern, inspect:

* **AuthPolicy** — combined OIDC and API Key authentication
* **RateLimitPolicy** — rate limits applied to traffic matching the HTTPRoute

[.console-input]
[source,bash]
----
oc get authpolicy -n %USER_NAME%-neuralbank -o yaml
oc get ratelimitpolicy -n %USER_NAME%-neuralbank -o yaml
----

== Summary

You correlated GitOps state in ArgoCD with catalog cards in Developer Hub, located the API and its OpenAPI, and placed authentication and rate limit policies in the Kuadrant context.
