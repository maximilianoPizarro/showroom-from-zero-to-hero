== What is DevSpaces?

*Red Hat DevSpaces* (formerly CodeReady Workspaces) provides cloud-based development environments that run directly on your OpenShift cluster. This means:

* *No Local Setup*: Everything runs in the cloud
* *Consistent Environment*: Same setup for all developers
* *Resource Efficiency*: Share cluster resources
* *Easy Collaboration*: Share workspaces with team members

For Kevin, DevSpaces means you can:

* Develop directly on OpenShift
* Access all cluster resources
* Test against real services
* Work with the generated `customer-service-mcp` code

== Accessing Your DevSpaces Workspace

The Golden Path template automatically creates a DevSpaces workspace configuration. Here's how to access it:

=== Step 1: Access from Developer Hub

The easiest way to open your DevSpaces workspace is directly from Developer Hub:

. *Navigate to Developer Hub*: https://backstage-developer-hub-backstage.apps.{cluster-domain}[backstage-developer-hub-backstage.apps.{cluster-domain}]
. *Find Your Component*: Look for the `customer-service-mcp` component
. *Open DevSpaces*: Click on the DevSpaces link or use the direct URL to open DevSpaces for customer-service-mcp

=== Step 2: Alternative Access Methods

You can also access DevSpaces through:

. *From OpenShift Console*: Go to *Applications* → *Red Hat DevSpaces*
. *Direct URL*: https://devspaces.apps.{cluster-domain}[devspaces.apps.{cluster-domain}]

=== Step 3: Open Your Workspace

. *Find Your Workspace*: Look for a workspace named `customer-service-mcp`
. *Click to Open*: This will launch your development environment

=== Step 4: Wait for Startup

The workspace will automatically:

* Clone the generated repository
* Start all required containers
* Configure the development environment
* Install necessary tools and dependencies

|===
|Feature |Description

|*Integrated Terminal*

|*Code Editor*
|Full IDE experience in the browser, IntelliSense and code completion, Git integration, extension support, multi-file editing

== Development Workflow

In your DevSpaces workspace:

. *Navigate to Code*: The generated `customer-service-mcp` code is already cloned
. *Review Structure*: Explore the generated code structure
. *Uncomment Code*: Find and uncomment the relevant sections (as mentioned in the template)
. *Build*: Run `mvn clean install` to build the project
. *Test*: Execute unit and integration tests
. *Run Locally*: Use Quarkus dev mode for local testing
. *Commit Changes*: Commit your uncommented code to start working

== Preview MCP Service

To preview and test the MCP service locally in your DevSpaces workspace:

. *Open Terminal*: Open a terminal in your DevSpaces workspace
. *Navigate to Project*:
+
[source,bash]
----
cd customer-service-mcp
----

. *Check README*: Review the README.md file for specific commands:
+
[source,bash]
----
cat README.md
----

image::devspaces-readme.png[DevSpaces README]

. *Run MCP Service*: Execute the commands from the README to start the MCP service in development mode. Typically:
+
[source,bash]
----
mvn quarkus:dev
----

. *Access MCP Endpoint*: The MCP service will be available at:
** *Endpoint*: `http://localhost:8081/mcp/sse`
** *Protocol*: HTTP
** *Port*: 8081

. *Test MCP Connection*: You can test the MCP endpoint using curl or MCP Inspector:
+
[source,bash]
----
curl http://localhost:8081/mcp/sse
----

.TIP
The MCP service uses Server-Sent Events (SSE) for real-time communication. Make sure to configure your MCP client (like Playground or Cursor) to use `http://localhost:8081/mcp/sse` when testing locally in DevSpaces.
