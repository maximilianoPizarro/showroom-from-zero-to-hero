You will create three components in sequence:

[cols="1,2,1"]

== Step 5: Wait for the Tekton Pipelines

The backend and MCP templates trigger a Tekton pipeline that:

. Clones the source code from Gitea
. Builds with Maven
. Creates a container image with Buildah
. Deploys to OpenShift

Check pipeline status:

[.console-input]
[source,bash]
----

----

Wait until the pipeline shows `Succeeded`. This usually takes 3-5 minutes.

== Step 6: Test the Backend API with curl

The pre-deployed **Neuralbank Stack** (in namespace `neuralbank-stack`) is protected with **OIDC** via Keycloak. To test it:

=== 6.1 — Get a Bearer Token from Keycloak

[.console-input]
[source,bash]
----
TOKEN=$(curl -s -X POST \
  "https://rhbk.apps.cluster-l9nhj.dynamic.redhatworkshops.io/realms/neuralbank/protocol/openid-connect/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=password" \
  -d "client_id=neuralbank-frontend" \
  -d "username=user1" \
  -d "password=Welcome123!" \
  | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")

echo "Token: ${TOKEN:0:50}..."
----

=== 6.2 — List all customers

[.console-input]
[source,bash]
----
curl -s -H "Authorization: Bearer $TOKEN" \
  "https://neuralbank.apps.cluster-l9nhj.dynamic.redhatworkshops.io/api/v1/customers" \
  | python3 -m json.tool | head -30
----

== Authentication Models Comparison

[cols="1,1,1"]
|===
| | OIDC (Neuralbank) | API Key (NFL Wallet)

| **How to authenticate**
| Get a JWT token from Keycloak, pass as `Authorization: Bearer <token>`
| Include a static key as `X-API-Key: <key>` header

| **Steps to test**
| 2 steps: get token + call API
| 1 step: call API with key in header

| **Best for**
| Web apps with user login
| Scripts, CI/CD, M2M integrations

| **Token expiry**
| Short-lived (minutes)
| No expiry (revoke by deleting Secret)
|===

TIP: For quick terminal testing, **API Key** is faster since it requires no token exchange. For production web apps, **OIDC** provides better security with short-lived tokens and user identity.
