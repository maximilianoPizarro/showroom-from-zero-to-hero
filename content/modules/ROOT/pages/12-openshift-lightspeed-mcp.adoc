= OpenShift Lightspeed & MCP Gateway

**OpenShift Lightspeed (OLS)** is an AI assistant integrated into the OpenShift web console. Combined with the **MCP Gateway** (a Kuadrant community extension), it can execute cluster operations through AI-callable tools.

== Accessing OpenShift Lightspeed

. Log in to the **OpenShift Console** at `https://console-openshift-console.__CLUSTER_DOMAIN__`
. Click the **Lightspeed** icon in the console header (chat bubble icon).
. Ask questions about your cluster, workloads, or Kubernetes concepts.

== Try These Prompts

[cols="1,2"]
|===
| Prompt | What Happens

| "What pods are running in my namespace?"
| OLS uses MCP tools to query pod status

| "Show me the gateways in neuralbank-stack"
| MCP calls `kubectl get gateway -n neuralbank-stack`

| "Explain the AuthPolicy in nfl-wallet-prod"
| OLS retrieves and explains the policy configuration

| "What events are in the %USER_NAME%-neuralbank namespace?"
| MCP queries cluster events for your namespace
|===

== OLS Configuration

OpenShift Lightspeed is configured through the `OLSConfig` custom resource:

[source,yaml]
----
apiVersion: ols.openshift.io/v1alpha1
kind: OLSConfig
metadata:
  name: cluster
spec:
  llm:
    providers:
      - name: vllm
        url: "https://vllm-endpoint.__CLUSTER_DOMAIN__/v1"
        type: openai
        credentialsSecretRef:
          name: vllm-credentials
        models:
          - name: qwen3-8b
    mcpServers:
      - name: k8s-mcp-server
        transport:
          type: sse
          url: "http://mcp-gateway.openshift-lightspeed.svc:3001/k8s-mcp-server/sse"
      - name: ocp-mcp-server
        transport:
          type: sse
          url: "http://mcp-gateway.openshift-lightspeed.svc:3001/ocp-mcp-server/sse"
----

== MCP Gateway Configuration

The MCP Gateway is deployed as a community extension of Kuadrant. It registers MCP servers that OpenShift Lightspeed can call:

[source,yaml]
----
apiVersion: gateway.kuadrant.io/v1alpha1
kind: MCPGatewayExtension
metadata:
  name: mcp-gateway
  namespace: openshift-lightspeed
spec:
  port: 3001
  replicas: 1
----

== MCP Server Registration

Each MCP server is registered with the gateway:

[source,yaml]
----
apiVersion: gateway.kuadrant.io/v1alpha1
kind: MCPServerRegistration
metadata:
  name: k8s-mcp-server
  namespace: openshift-lightspeed
spec:
  name: k8s-mcp-server
  description: "Kubernetes MCP server for cluster operations"
  upstream:
    url: "http://k8s-mcp-server.openshift-lightspeed.svc:3001/sse"
---
apiVersion: gateway.kuadrant.io/v1alpha1
kind: MCPServerRegistration
metadata:
  name: ocp-mcp-server
  namespace: openshift-lightspeed
spec:
  name: ocp-mcp-server
  description: "OpenShift MCP server for cluster operations"
  upstream:
    url: "http://ocp-mcp-server.openshift-lightspeed.svc:3001/sse"
----

== What is MCP (Model Context Protocol)?

MCP is an open standard that allows AI models to interact with external tools securely:

* **Tools** — Functions the AI can call (e.g., `kubectl get pods`)
* **Resources** — Data the AI can read (e.g., cluster state)
* **Prompts** — Pre-built conversation templates

The MCP Gateway acts as a proxy between OLS and individual MCP servers, providing unified access and security.

== Customer Service MCP

The workshop also deploys a **Customer Service MCP** as part of the Neuralbank stack. This MCP server exposes backend operations (customer CRUD, credit scoring) as AI-callable tools:

[.console-input]
[source,bash]
----
# Check MCP server tools endpoint
curl -s -H "X-API-Key: $API_KEY" \
  "https://%USER_NAME%-customer-service-mcp.__CLUSTER_DOMAIN__/tools" \
  | python3 -m json.tool
----

This demonstrates how MCP can bridge business applications with AI assistants.

== Summary

OpenShift Lightspeed with the MCP Gateway transforms the OpenShift console into an AI-powered operations center. The MCP protocol allows the AI to safely execute cluster operations while the gateway provides unified access control through Kuadrant.
